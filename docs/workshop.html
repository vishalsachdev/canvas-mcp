Canvas MCP + Codex

Set up AI-powered Canvas LMS tools in under 2 minutes. No coding required.

1 Download the Codex App 1 min

Download the Codex desktop app for macOS:

Download Codex for Mac

  1. Open the .dmg file and drag Codex to Applications
  2. Launch Codex and sign in with your ChatGPT account

Download the Codex desktop app for Windows:

Download Codex for Windows

  1. Install from the Microsoft Store
  2. Launch Codex and sign in with your ChatGPT account
You need a ChatGPT Plus, Team, or Enterprise account. Free accounts don't include Codex.

2 Get Your Canvas API Token 1 min

  1. Log into Canvas in your browser
  2. Click your profile picture (top-left) → Settings
  3. Scroll down to Approved Integrations
  4. Click + New Access Token
  5. Set purpose to Canvas MCP, leave expiry blank, click Generate Token
  6. Copy the token now — you won't be able to see it again
Save the token somewhere safe (e.g., a notes app). You'll need it in the next step.

3 Run the Setup Wizard 30 sec

Open your terminal and run:

$ npx canvas-mcp setup

The wizard will ask you three things:

  1. Canvas API token — paste the token from Step 2
  2. Canvas URL — your instructor will provide this
  3. Which client — select Codex CLI (press Space, then Enter)
Important: After the wizard finishes, it will tell you to set an environment variable. Run the command it shows:
$ export CANVAS_API_TOKEN="your-token-here"
No Node.js? Download it from nodejs.org (LTS version). Then re-run the command above.

4 Restart Codex and Test 30 sec

  1. Quit Codex completely (Cmd+Q on Mac, close window on Windows)
  2. Reopen Codex
  3. Open a new session and try this prompt:
What courses am I enrolled in?
If you see your course list, you're all set!

Troubleshooting

npx command not found

You need Node.js installed. Download from nodejs.org (LTS version), then restart your terminal and try again.

Codex doesn't see Canvas tools after restart

Make sure you set the environment variable. Open a new terminal and run:

$ export CANVAS_API_TOKEN="your-token"

Then launch Codex from that same terminal, or add the export line to your ~/.zshrc (Mac) or ~/.bashrc (Linux/Windows WSL).

Canvas API token doesn't work

Double-check that you copied the entire token (they're long). Also verify the Canvas URL matches exactly what your instructor provided (no trailing slash).

I see "unauthorized" or "forbidden" errors

Your Canvas token may have expired or been revoked. Go back to Canvas Settings → Approved Integrations and generate a new token. Then re-run npx canvas-mcp setup.
